# 🎬 Media Tools Web Workspace

> 一个围绕 **素材获取 → 文稿生成 → 结果管理** 组织的本地内容工作台。
>
> 当前主入口：**Streamlit Web 面板**（分支：`feature/streamlit-web`）。

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE.txt)
[![Python Version](https://img.shields.io/badge/python-3.11%2B-brightgreen.svg)](https://www.python.org/)

---

## 新版本定位

Media Tools 现在不再强调“全自动神器”式表达，而是明确定位为：

**一个帮助你把视频素材更快转成文稿，并在本地完成管理与整理的内容工作台。**

这意味着项目的主线很清楚：

1. **获取素材**：从抖音链接或来源列表拉取视频
2. **生成文稿**：把视频 / 音频转写成可整理的文字稿
3. **管理结果**：查看任务、维护来源、管理空间、检查配置

---

## 适合谁使用

这个版本更适合下面这些场景：

- 自媒体创作者：把视频快速转成文字稿，便于整理和二次创作
- 内容研究 / 竞品分析：按来源批量收集素材，统一管理
- 本地资料整理：把分散的视频转成结构化文稿并分类保留
- 需要浏览器操作而非命令行的用户

## 当前版本的核心价值

### 1. 素材获取
- 粘贴单个抖音链接下载视频
- 从关注来源列表批量拉取素材
- 在素材库中查看最近入库结果

### 2. 文稿生成
- 上传单个本地视频 / 音频文件转写
- 直接处理 `downloads/` 中的素材
- 在文稿库中查看最近生成结果

### 3. 工作台管理
- 在工作台查看系统状态、最近任务和存储占用
- 管理来源列表、账号与配额、清理与备份、系统配置
- 通过统一的 Web 结构完成主要操作路径

## 与旧版本相比，有什么变化

这个分支最重要的变化不是“又多了几个功能”，而是：

### 以前
- README 更像功能堆叠页
- 页面命名和功能表达不够统一
- 用户需要自己理解“下载 / 转写 / 管理”之间的关系

### 现在
- 整体按工作流组织
- 页面围绕 **素材 / 文稿 / 来源** 三类资源设计
- 工作台负责“看状态 + 决定下一步”
- 摘要区、高亮卡片、空状态、表格区开始组件化

## 当前 Web 结构

```text
🏠 工作台
  ├── 📥 下载中心
  ├── 🎙️ 转写中心
  ├── 👥 关注管理
  ├── 🔑 账号与配额
  ├── 🗑️ 清理与备份
  └── ⚙️ 系统配置
```

### 1. `🏠 工作台`
用于先确认系统状态，再决定下一步。

包含：
- 系统状态卡片
- 推荐下一步
- 常用入口
- 最近任务
- 存储概览

### 2. `📥 下载中心`
围绕“把链接或来源变成素材”组织。

包含：
- 创建任务
- 当前任务
- 素材库

### 3. `🎙️ 转写中心`
围绕“把素材变成文稿”组织。

包含：
- 发起转写
- 当前任务
- 文稿库

### 4. `👥 关注管理`
围绕“维护来源列表”组织。

包含：
- 来源列表
- 添加来源
- 导入 / 导出

### 5. `🔑 账号与配额`
用于检查转写能力是否可用。

### 6. `🗑️ 清理与备份`
用于回收空间、清理旧记录和备份关键数据。

### 7. `⚙️ 系统配置`
用于环境检测、配置修复和预设应用。

## 推荐使用方式

### Web 模式（推荐）

```bash
cd /path/to/media-tools
pip install -e .
pip install streamlit
playwright install chromium
media-tools-web
```

启动后访问：`http://localhost:8501`

### CLI 模式（补充入口）

```bash
cd /path/to/media-tools
media-tools-cli
```

CLI 仍可用，但当前分支默认推荐 Web 作为主入口。

## 快速开始

### 1. 准备配置文件

```bash
cp config/config.yaml.example config/config.yaml
cp config/transcribe/.env.example config/transcribe/.env
```

### 2. 完成关键认证

需要至少准备两类认证：

- **抖音 Cookie**：用于下载素材
- **Qwen 认证**：用于转写生成文稿

推荐优先在 Web 中完成检查：

- `⚙️ 系统配置`：环境检测、下载侧配置
- `🔑 账号与配额`：转写认证与额度检查

### 3. 按实际工作流验证

推荐第一次按这个顺序走：

```text
工作台
  ↓
系统配置
  ↓
下载中心
  ↓
转写中心
  ↓
文稿库 / transcripts/
```

最短验证路径：

1. 打开 `🏠 工作台`
2. 去 `⚙️ 系统配置` 做环境检测
3. 去 `📥 下载中心` 获取一个样例视频
4. 去 `🎙️ 转写中心` 生成文稿
5. 在 `📝 文稿库` 中确认结果
