# 设计规范：全平台认证看板化重构

## 1. 目标
将原先割裂的 3 个 Tab（账号列表、配额查询、认证配置）整合为**按平台维度**的卡片式看板。
解决“用户不知道系统当前哪些平台正常、哪些失效”以及“配额和账号管理不在一个地方”的槽点。

## 2. 界面结构设计 (Mac 极简风格)

### 2.1 页面头部
- 标题：`🔑 账号与认证`
- 描述：`管理系统依赖的所有平台账号与 Cookie 授权。`

### 2.2 平台卡片：🎵 抖音 (Douyin)
使用 `st.container(border=True)` 包裹，保持毛玻璃卡片质感。
- **头部**：左侧图标+名称，右侧使用 Tag 显示状态 (`✅ 正常` 或 `❌ 未认证`)。
- **信息区** (2列布局)：
  - 左列：认证方式 (Cookie)、存储位置 (ConfigManager)。
  - 右列：目前使用的 Cookie 预览 (脱敏显示前20个字符)。
- **操作区** (使用 `st.expander("更新认证")` 或直接展开)：
  - 引导文本：提供如何获取 Cookie 的帮助说明。
  - 输入框：`st.text_area` 接收新 Cookie。
  - 按钮：`[保存抖音认证]` (Primary)。

### 2.3 平台卡片：🤖 通义千问 (Qwen)
使用 `st.container(border=True)` 包裹。
- **头部**：左侧图标+名称，右侧 Tag 显示状态 (`✅ 正常` 或 `❌ 未认证`)。
- **配额速览区** (只有状态为正常时才显示)：
  - 使用 3 个小 `st.metric` 并排展示：`总配额`、`已使用`、`剩余分钟`。
  - 获取配额逻辑直接在卡片渲染时异步/同步调用，不需用户手动点击。
- **信息区**：
  - 认证方式 (Playwright State)。
  - 多账号池：读取 `accounts.json`，如果有多个账号，显示列表预览（如“检测到 2 个备用账号”）。
- **操作区**：
  - 引导文本：提供如何获取 Cookie 的帮助说明。
  - 输入框：`st.text_area` 接收新 Cookie。
  - 按钮：`[保存 Qwen 认证]` (Primary)。

## 3. 代码重构策略

1. **废弃 Tab**：删除 `st.tabs` 逻辑。
2. **状态检测函数提取**：
   - 提取 `_check_douyin_status() -> bool`
   - 提取 `_check_qwen_status() -> bool`
3. **配额自动加载**：将 `_get_quota` 从手动触发改为卡片内部的状态驱动加载（使用 `st.spinner` 或空状态占位，避免阻塞主线程太久，但因为是本地环境，通常很快）。
4. **组件拆分**：
   - `_render_douyin_card()`
   - `_render_qwen_card()`
5. **样式优化**：结合现有的 Mac 风格 `theme.py`，卡片内使用 `st.divider()` 和 `st.columns` 梳理层级，避免使用过多的强警告框（`st.error`/`st.warning`），改用更温和的文本或自定义 HTML 标签表示状态。
