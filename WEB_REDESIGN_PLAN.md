# Media Tools Web 管理面板 - 全面重设计规划

> **目标**: 从用户角度重新规划整个 Web 应用，打造专业、直观、易用的媒体管理面板
> **创建时间**: 2026-04-13
> **设计原则**: 用户为中心、任务导向、视觉清晰、操作流畅

---

## 一、当前问题分析

### 1.1 信息架构问题
- **7 个页面平铺**，没有主次关系，用户不知道从哪里开始
- **功能分散**：相关功能分布在不同页面（如下载和转写应该关联）
- **缺乏主页概念**：侧边栏直接进入功能页，没有总览和引导

### 1.2 用户流程问题
- **新手困惑**：虽然有引导，但进入后仍不知道各功能关系
- **任务不完整**：下载 → 转写 → 管理 是完整流程，但现在割裂
- **反馈不及时**：操作后需要手动刷新或跳转才能看到结果

### 1.3 视觉设计问题
- **Streamlit 默认风格**，缺乏品牌感
- **信息密度不均**：有的页面太空，有的太挤
- **缺少图标和视觉提示**：纯文字导航不够直观

---

## 二、重设计方案

### 2.1 新的信息架构

```
🏠 首页（总览仪表盘）
  ├── 系统状态卡片（认证、环境、存储）
  ├── 快速操作区（最常用的 4 个操作）
  ├── 最近任务历史（最近 5 个任务）
  └── 数据统计概览（图表展示）

📥 下载中心
  ├── 单链接下载
  ├── 批量下载关注
  ├── 下载任务队列（当前/历史）
  └── 下载管理（已下载视频列表、预览）

🎙️ 转写中心
  ├── 单文件转写
  ├── 批量转写
  ├── 转写任务队列
  └── 文稿管理（已转写列表、预览、下载）

👥 关注管理
  ├── 关注列表（表格视图）
  ├── 添加关注（URL/批量导入）
  └── 关注统计（粉丝数、视频数趋势）

⚙️ 系统设置
  ├── 环境检测
  ├── Cookie/认证管理
  ├── 配置管理（预设、备份、恢复）
  └── 系统信息（版本、日志、帮助）
```

### 2.2 关键改进点

#### 改进 1: 创建真正的"首页"
- **目的**: 用户一进来就知道系统状态和能做什么
- **内容**:
  - 系统健康状态（绿灯/黄灯/红灯）
  - 快速操作按钮（根据当前状态动态显示）
  - 最近任务历史（一看就知道刚才做了什么）
  - 存储使用图表（视觉化展示空间占用）

#### 改进 2: 下载和转写整合为"中心"
- **目的**: 反映真实工作流（下载 → 转写 → 管理）
- **设计**:
  - 每个中心有"新建任务"和"任务管理"两个标签
  - 任务历史在同一页面，不用跳转
  - 下载完成后可直接点击"转写此文件"

#### 改进 3: 任务系统升级
- **当前**: 单任务，JSON 文件轮询
- **改进**:
  - 支持任务队列（多个任务排队）
  - 任务详情页面（查看完整日志）
  - 任务分类（下载/转写/清理）
  - 任务筛选（全部/成功/失败/运行中）

#### 改进 4: 关注管理增强
- **新增**:
  - 关注博主的视频数/粉丝数趋势图
  - 按最后下载时间排序
  - 一键下载某个博主的更新
  - 关注分组（可选功能）

#### 改进 5: 设置页面专业化
- **重新组织**:
  - 环境检测（独立卡片）
  - 认证管理（Cookie + Qwen 账号）
  - 配置管理（预设模板、导入导出）
  - 系统信息（版本、日志、帮助链接）

### 4.2 核心改进实现

#### 改进 A: 任务队列升级
- 当前: 单任务，覆盖状态文件
- 改进: 多任务，每个任务独立状态文件（`.tasks/task_{id}.json`）
- 好处: 支持并发、排队、历史查询

#### 改进 B: 首页实现
- 整合仪表盘、统计、任务历史
- 使用卡片布局，视觉清晰
- 动态内容（根据系统状态显示不同提示）

#### 改进 C: 下载/转写中心
- 每个中心 3 个标签：新建、队列、管理
- 任务状态实时更新
- 下载完成后可直接转写

## 六、视觉设计指南

### 6.1 色彩方案
- **主色**: 蓝色 (#1E88E5) - 专业、可信赖
- **成功**: 绿色 (#43A047)
- **警告**: 橙色 (#FB8C00)
- **错误**: 红色 (#E53935)
- **信息**: 灰色 (#757575)

### 6.2 卡片设计
- 圆角、阴影
- 图标 + 标题 + 内容
- 悬停效果（如果 Streamlit 支持）

### 6.3 布局原则
- 重要内容在上，操作在下
- 相关功能分组
- 留白充足，不拥挤

## 七、用户故事

### 故事 1: 新用户首次使用
1. 访问 Web 界面
2. 看到新手引导（4 步）
3. 完成引导后进入首页
4. 首页显示系统状态和快速操作
5. 点击"下载"开始第一个任务

### 故事 2: 日常使用
1. 访问首页，查看系统状态
2. 点击"下载中心"
3. 创建批量下载任务
4. 在任务队列查看进度
5. 下载完成后在"已下载管理"预览
6. 选择视频点击"转写"
7. 在转写中心查看转写进度
8. 下载转写后的文稿

### 故事 3: 管理关注列表
1. 进入"关注管理"
2. 搜索博主或添加新关注
3. 按粉丝数排序找到热门博主
4. 勾选多个博主点击"批量下载"
5. 返回下载中心查看进度
