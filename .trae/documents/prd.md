## 1. 产品概述
抖音视频数据可视化面板是一个纯前端的静态单页应用（SPA），旨在帮助用户以极高的审美标准和流畅的交互体验，浏览、搜索和分析通过爬虫工具下载到本地的抖音博主视频及相关统计数据。
- 解决本地下载视频后难以进行数据对比、排序和直观预览的问题。
- 目标用户为自媒体从业者、竞品分析人员及短视频创作者。
- 核心价值：将冷冰冰的本地 `.mp4` 文件和 `sqlite` 数据转化为美观、现代、极具洞察力的数据看板。

## 2. 核心功能

### 2.1 功能模块
1. **全局统计模块 (Global Stats)**：展示收录的博主总数、视频总数、累计点赞量、占用磁盘空间及最后更新时间。
2. **高级过滤与排序栏 (Filter & Sort Bar)**：
   - 实时文本搜索：基于视频标题和文案的毫秒级全文检索。
   - 多维度排序：支持按点赞量、评论量、发布时间（正序/倒序）、文件大小进行动态重排。
3. **博主聚合卡片 (Author Cards)**：
   - 展示博主昵称、本地视频数量、获赞总数。
   - 采用 Accordion（手风琴）交互，点击展开/折叠该博主名下的所有视频瀑布流。
4. **视频数据卡片 (Video Cards)**：
   - 视频封面海报预览（读取本地同名 `.jpg`）。
   - 内置 HTML5 视频播放器，悬停或点击即可播放。
   - 截断式标题展示（防长文案破坏排版）。
   - 核心指标（点赞、评论、文件大小）的微型数据条。

## 3. 核心流程
1. 用户在文件管理器中双击打开 `index.html`。
2. 页面加载内置的 `data.js` 静态数据变量 (`window.APP_DATA`)。
3. JavaScript 初始化渲染引擎，计算全局统计数据并渲染博主列表。
4. 用户在搜索框输入关键字，触发 `handleSearch` 事件。
5. 系统根据关键字和当前选择的 `sortType` 重新过滤并排序 `data.videos`。
6. 触发 `renderAuthors()`，DOM 局部更新，符合条件的博主卡片自动展开 (`active`) 显示匹配的视频。

## 4. 界面设计 (UI/UX)

### 4.1 设计风格 (Glassmorphism + Dark Mode)
我们承诺采用**极具质感的深色毛玻璃 (Glassmorphism)** 美学方向，摒弃廉价的后台管理系统既视感。
- **色彩 (Color)**：
  - 背景底色：极深蓝灰 (`#0f172a`，Slate 900)
  - 卡片底色：半透明幽灵白 (`rgba(30, 41, 59, 0.7)`)，配合 `backdrop-filter: blur(10px)` 营造毛玻璃纵深感。
  - 强调色 (Accent)：电光蓝 (`#3b82f6`) 与 紫光渐变 (`#a78bfa`)。
  - 文本色：主文本 `#f8fafc`，次要信息 `#94a3b8`。
- **字体 (Typography)**：
  - 采用系统级无衬线优雅字体栈 (`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto...`)。
  - 大标题采用渐变遮罩 (`-webkit-background-clip: text`) 呈现未来科技感。
- **动效 (Motion)**：
  - 悬停浮动：卡片 `hover` 时 `transform: translateY(-2px)` 配合柔和的阴影扩散。
  - 展开动画：`@keyframes fadeIn` 确保视频网格展开时由下至上平滑淡入。
- **布局 (Spatial Composition)**：
  - 顶部留白充足，卡片间距 `24px`。
  - 视频列表采用自适应 Grid 网格 (`minmax(280px, 1fr)`)。

### 4.2 响应式设计
- **Desktop First**：专为宽屏数据分析场景设计，最大容器宽度 `1200px` 居中。
- 视频网格使用 CSS Grid 自动换行，确保在移动端（单列）或超宽屏（多列）下均能保持优雅比例。
- 搜索框与下拉菜单在移动端自动折叠为块级元素。
