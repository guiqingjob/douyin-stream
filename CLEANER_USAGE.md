# 数据清理功能使用说明

## 问题背景

之前删除本地视频文件后,数据库中的记录仍然存在,导致系统会误判为"有新视频"并重新下载。

## 新增功能

### 1. 智能清理已删除视频记录

**功能说明:**
- 自动扫描本地视频文件
- 对比数据库记录
- 清理已删除视频的元数据记录
- 避免重复下载已删除的视频

**使用方法:**
1. 运行 `python cli.py`
2. 选择 `8. 🗑️  数据清理`
3. 选择 `1. 清理已删除视频的数据库记录`
4. 系统会自动检测并清理多余记录

**工作流程:**
```
本地文件: 100个视频
数据库记录: 120条记录
→ 检测到差异: 20条
→ 清理多余的20条数据库记录
→ 保持同步,不会重复下载
```

### 2. 清理指定博主的所有数据

**功能说明:**
- 清理指定博主的所有数据库记录
- 包括视频元数据和用户信息
- 适合完全不想再跟踪的博主

**使用方法:**
1. 运行 `python cli.py`
2. 选择 `8. 🗑️  数据清理`
3. 选择 `2. 清理指定博主的所有数据库记录`
4. 选择要清理的博主
5. 确认后清理

### 3. 重新下载所有视频

如果您想要**重新下载所有视频**(包括之前删除的):

**方法1: 清理数据库后重新下载**
1. 先运行 `8. 🗑️  数据清理` → 清理指定博主的所有数据
2. 然后运行 `2. 📥 下载所有更新`
3. 系统会认为所有视频都是"新的",全部重新下载

**方法2: 删除数据库文件**
```bash
# 删除整个数据库(谨慎操作!)
rm downloads/f2.db
```
然后重新运行下载功能,系统会重新获取所有数据。

## 建议工作流

### 日常删除视频后
```
1. 删除本地视频文件
2. 运行 "8. 🗑️  数据清理" → "1. 清理已删除视频的数据库记录"
3. 系统保持同步,不会重复下载
```

### 想要重新下载某个博主
```
1. 运行 "8. 🗑️  数据清理" → "2. 清理指定博主的所有数据库记录"
2. 选择该博主
3. 运行 "2. 📥 下载所有更新" 或 "4. 📺 视频下载"
4. 系统会重新下载所有视频
```

## 技术实现

新增模块: `scripts/core/cleaner.py`

主要函数:
- `scan_local_videos()` - 扫描本地视频文件
- `get_db_video_records()` - 获取数据库记录
- `clean_deleted_videos()` - 清理已删除视频的记录
- `clean_all_user_data()` - 清理用户所有数据

## 注意事项

1. **清理操作可逆**: 清理的只是本地数据库记录,不影响抖音服务器上的视频
2. **定期清理**: 建议定期(如每周)运行一次清理功能,保持数据同步
3. **备份数据库**: 大量清理前可以备份 `downloads/f2.db` 文件
