# Media Tools Web UI Guide

> 用于约束 `feature/streamlit-web` 分支后续页面演进的轻量设计规范。

---

## 1. 页面结构模板

建议所有核心页面按以下顺序组织：

1. **页面标题**
2. **一句话目的说明**
3. **摘要指标区**
4. **高亮结果 / 当前重点信息**
5. **表格区或任务区**
6. **空状态或下一步提示**

典型对应：

- 下载中心：任务创建 → 当前任务 → 素材库
- 转写中心：任务创建 → 当前任务 → 文稿库
- 关注管理：来源列表 → 添加来源 → 导入导出

---

## 2. 通用 UI 组件

统一优先复用：

- `render_summary_metrics(...)`
  - 用于摘要区
- `render_highlight_card(...)`
  - 用于最近结果 / 重点对象
- `render_empty_state(...)`
  - 用于空状态
- `render_table_section(...)`
  - 用于数据表格区

原则：

- 不要在每个页面重复手写一套 `metric + container + dataframe + info`
- 优先复用公共模式，再补局部差异

---

## 3. 按钮命名规范

### 3.1 主执行动作
统一使用：

- `🚀 开始下载`
- `🚀 开始转写`
- `🚀 开始批量拉取`
- `🚀 开始批量转写`

规则：
- 主动作尽量使用“开始 + 动作”
- 避免同类页面里同时出现“提交 / 执行 / 运行 / 启动”多套说法

### 3.2 页面跳转动作
统一使用：

- `🎙️ 去转写中心`
- `📥 去下载中心批量拉取`

规则：
- 跨页面下一步，尽量使用“去 + 页面/目标动作”

### 3.3 导入导出动作
统一使用：

- `📥 下载 JSON`
- `📤 开始导入`

规则：
- 导出用“下载”
- 导入用“开始导入”

---

## 4. 空状态文案规范

空状态统一为两层：

1. **当前状态是什么**
2. **下一步建议是什么**

示例：

- `素材库为空。`
- `先去左侧创建一个下载任务，拿到第一批素材。`

避免：

- 只有“暂无数据”但不给下一步
- 用过多技术细节做空状态主文案

---

## 5. 表格区规范

表格区建议结构：

1. 摘要区（如有）
2. 高亮结果（如有）
3. 表格
4. 下一步提示

表格字段命名尽量使用业务语言，例如：

- `文件名`
- `修改时间`
- `最近生成`
- `来源数量`

避免把底层字段名直接暴露给用户。

---

## 6. 当前任务区规范

当前任务区建议包含：

- 任务类型
- 当前状态
- 当前进度
- 任务说明
- 结果摘要 / 取消动作

统一优先复用 `render_task_progress(...)`。

---

## 7. 文案语气原则

建议保持：

- 直接
- 简洁
- 像技术产品，不像营销页

推荐：

- “把素材变成文稿”
- “用于后续批量拉取素材”
- “先执行一次任务再回来查看结果”

避免：

- “神器”
- “极致”
- “一键搞定所有事”
- 过度承诺式表达

---

## 8. 后续演进建议

后续如果继续扩展页面：

- 先判断能否复用现有 UI 模式组件
- 新增模式前，优先检查是否只是现有组件参数不够
- 只有当页面结构真的不同，才新增新组件
