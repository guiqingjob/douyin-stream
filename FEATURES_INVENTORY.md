# Media Tools 全部功能清单

## 一、创作者管理（Creators）

| # | 功能 | 说明 |
|---|------|------|
| 1 | 添加创作者 | 粘贴抖音/B站主页链接，解析博主信息并加入关注列表 |
| 2 | 查看创作者列表 | 显示所有关注的博主，含昵称、头像、平台、同步状态 |
| 3 | 删除创作者 | 从关注列表移除博主，可选择是否删除已下载的视频 |
| 4 | 创作者全量同步 | 下载该博主的所有视频（忽略已下载的），下载完自动转写 |
| 5 | 创作者增量同步 | 只下载该博主在上次同步之后发布的新视频，自动转写 |

## 二、素材管理（Assets）

| # | 功能 | 说明 |
|---|------|------|
| 6 | 查看素材列表 | 按创作者分组显示已下载的视频 |
| 7 | 查看转写内容 | 点击视频查看对应的 markdown 转写文稿 |
| 8 | 搜索素材 | 搜索视频标题或转写内容的关键词 |
| 9 | 导出转写 | 批量选择视频，导出为 zip 格式的 markdown 文件 |
| 10 | 删除素材 | 删除单个视频及其转写文件 |
| 11 | 标记已读 | 标记视频为已读状态 |
| 12 | 收藏素材 | 标记视频为收藏状态 |
| 13 | 批量标记 | 批量标记多个视频为已读/收藏 |
| 14 | 批量删除 | 批量删除多个视频 |
| 15 | 清理素材 | 清理数据库中记录存在但文件已被删除的素材 |

## 当前问题分类

### 功能相关的问题

| # | 问题 | 影响 |
|---|------|------|
| 1 | 自动转写设置不同步 | 前端开启自动转写，但后端不执行 |
| 2 | 增量同步逻辑不明确 | 用户不清楚增量同步是怎么判断"新视频"的 |
| 3 | 任务状态显示不清晰 | 任务中心只显示进度数字，不显示当前阶段 |
| 4 | 全量同步和增量同步区分不明显 | 两个按钮功能相似，用户容易混淆 |
| 5 | 多账号管理复杂 | 抖音/B站/Qwen 各有多个账号，管理混乱 |

### 代码相关的问题

| # | 问题 | 影响 |
|---|------|------|
| 6 | 配置系统三源鼎立 | SystemSettings 表、config.yaml、环境变量不一致 |
| 7 | 348 处直接数据库访问 | 19 个文件直接操作数据库，没有统一入口 |
| 8 | tasks.py 1735 行 | 一个文件做了太多事，难以维护 |
| 9 | 没有分层架构 | API/业务逻辑/数据访问混在一起 |
| 10 | 前端组件过大 | Settings.tsx 798 行、TaskMonitorPanel.tsx 521 行 |

## 优化重构建议（保持所有功能）

### 优先级 1：修复当前 Bug

1. **统一配置系统**
   - 把 config.yaml 的内容迁移到 SystemSettings 表
   - 所有代码统一从数据库读取配置
   - 删除 config.yaml

2. **修复自动转写**
   - 确保前端设置页面的"自动转写"开关能正确传递到后端 Worker
   - 当前问题：前端保存到 SystemSettings，后端读取 config.yaml

### 优先级 2：改善用户体验

3. **任务中心增强**
   - 显示当前阶段（下载中 / 转写中 / 已完成）
   - 显示成功/失败数量统计
   - 显示每个子任务的状态

4. **增量同步逻辑明确化**
   - 显示"上次同步时间"
   - 增量同步前预览：将下载 X 个新视频
   - 同步完成后显示：本次新增 X 个，转写成功 Y 个

5. **简化多账号管理**
   - 每个平台只保留"主账号"概念
   - 多账号轮询作为高级选项，默认隐藏

### 优先级 3：代码架构改善

6. **添加 Repository 层**
   - 把 348 处数据库访问集中到 4 个 Repository 文件
   - API Router 只调用 Repository，不写 SQL

7. **拆分 tasks.py**
   - 把 1735 行拆分为：API Router + Service + Workers
   - API Router 只处理 HTTP 请求
   - Service 处理业务逻辑
   - Workers 处理后台任务

8. **拆分前端组件**
   - Settings.tsx 拆分为多个子组件
   - TaskMonitorPanel.tsx 拆分为多个子组件

### 优先级 4：可维护性

9. **添加集成测试**
   - 测试完整的"添加创作者 → 同步 → 转写"流程
   - 测试配置修改后能正确生效

10. **统一错误处理**
    - 定义标准的错误格式
    - 前端统一显示错误信息
    - 后端统一记录错误日志
