# Media Tools 用户场景功能测试方案

> 基于真实用户场景的完整功能测试方案
> 项目：抖音下载 + AI 智能转写一体化 CLI 工具
> 日期：2026-04-12

---

## 一、用户画像分析

### 角色 1：自媒体创作者（核心用户）

**背景**：抖音内容创作者，需要定期将关注博主的视频下载到本地，并对有价值的视频进行 AI 转写，用作文案参考、内容整理和二次创作素材。

**核心诉求**：
- 一键 Pipeline 下载并自动转写，省时省力
- 转写后自动清理本地视频，释放硬盘空间
- 关注列表管理，持续维护关注的博主
- 数据看板，分析博主互动数据（点赞、评论等）

**技术能力**：中等，会使用命令行，但需要清晰的菜单引导和错误提示。

---

### 角色 2：运营人员（批量处理用户）

**背景**：MCN 机构或内容团队的运营人员，需要批量管理多个博主账号，定期下载大量视频并进行转写分析，用于竞品监控和内容策略。

**核心诉求**：
- 批量下载和视频压缩，处理大量视频
- 多账号轮换，配额管理
- 增量更新，只处理新视频，避免重复操作
- 批量转写，高并发处理

**技术能力**：中低，偏好交互式菜单，需要自动化程度高。

---

### 角色 3：数据研究者（分析型用户）

**背景**：高校研究员或数据分析师，需要抓取抖音视频及其元数据（点赞、评论、收藏等），用于学术研究或市场分析。

**核心诉求**：
- 全量批量下载，获取完整数据集
- 数据库元数据完整准确
- 数据看板生成，可视化分析
- 关注列表的批量导入/导出

**技术能力**：较高，理解命令行和数据库，可能直接使用底层脚本。

---

### 角色 4：内容审核/存档人员（合规用户）

**背景**：需要定期存档特定博主的视频内容，保留转写文稿作为记录，对数据的完整性和可追溯性有较高要求。

**核心诉求**：
- 增量更新，定期拉取新内容
- 保留原始文件，不自动删除
- 转写文稿的多格式输出（MD + DOCX）
- 日志和状态记录完整

**技术能力**：中低，需要简单直观的操作流程。

---

## 二、用户旅程地图

### 旅程 1：自媒体创作者 — 从零到首次产出

| 阶段 | 操作 | 涉及功能 | 预期体验 |
|------|------|----------|----------|
| 1. 安装 | `pip install -r requirements.txt` + `playwright install chromium` | 环境检测（菜单 11→1） | 环境检测自动提示缺失依赖 |
| 2. 配置 | 复制配置模板 `cp config/config.yaml.example config/config.yaml` 等 | 配置管理（菜单 11→4） | 配置模板完整，有注释 |
| 3. 抖音认证 | 菜单 11→2，扫码登录 | 抖音扫码登录 | Playwright 打开浏览器，扫码后自动保存 Cookie |
| 4. Qwen 认证 | 菜单 9→1，扫码登录 Qwen AI | 转写认证管理 | 浏览器打开，登录成功保存 auth state |
| 5. 添加博主 | 菜单 3→2，粘贴抖音主页链接 | 关注列表管理 | 自动提取 sec_user_id，显示博主昵称 |
| 6. 首次 Pipeline | 菜单 5→1，输入单个抖音视频链接 | Pipeline 全自动转写 | 下载→转写→文稿输出，一气呵成 |
| 7. 验证结果 | 查看 `transcripts/` 目录 | 转写文稿查看 | 文稿含标题、说话人、时间戳、完整内容 |

### 旅程 2：运营人员 — 日常批量处理

| 阶段 | 操作 | 涉及功能 | 预期体验 |
|------|------|----------|----------|
| 1. 启动检查 | `python cli.py`，启动时自动检查更新 | 启动时更新检查 | 提示有几位博主更新了内容 |
| 2. 检查更新 | 菜单 1 | 检查博主更新 | 列出每位博主的新视频数量 |
| 3. 下载更新 | 菜单 2 | 下载所有更新 | 批量下载，显示进度 |
| 4. 批量 Pipeline | 菜单 5→2，从关注列表批量下载并转写 | Pipeline 批量模式 | 设置每位博主下载数量，并发执行 |
| 5. 视频压缩 | 菜单 6 | 视频压缩 | FFmpeg 压缩，节省空间 |
| 6. 查看看板 | 菜单 7 | 数据看板生成 | 生成 `data.js` + `index.html` |
| 7. 清理 | 菜单 12 | 数据清理 | 清理过期/无用数据 |

### 旅程 3：数据研究者 — 全量数据获取

| 阶段 | 操作 | 涉及功能 | 预期体验 |
|------|------|----------|----------|
| 1. 批量导入关注 | 菜单 3→5 | 批量导入关注列表 | 支持 JSON 批量导入 |
| 2. 全量下载 | 菜单 4 | 视频下载（全量模式） | 下载所有博主全部视频 |
| 3. 验证数据库 | 检查 SQLite 数据库 | 元数据入库 | 点赞/评论/收藏/分享数据完整 |
| 4. 生成看板 | 菜单 7 | 数据看板 | Web 面板展示统计信息 |
| 5. 数据导出 | 菜单 3→4 | 导出关注列表 | 导出为 JSON |

### 旅程 4：遇到问题时的处理路径

| 场景 | 用户操作 | 涉及功能 | 预期体验 |
|------|----------|----------|----------|
| Cookie 过期 | 菜单 11→2，重新扫码 | 抖音扫码登录 | 覆盖旧 Cookie，无残留 |
| 转写配额不足 | 菜单 10 | 账号状态/配额 | 显示剩余额度，引导领取 |
| 转写失败 | 查看日志 + 错误类型 | Pipeline 重试机制 | 区分网络/配额/认证错误，自动重试 |
| 磁盘空间不足 | 菜单 12 | 数据清理 | 列出可清理项，确认后删除 |
| 关注列表混乱 | 菜单 3→3 | 删除关注 | 搜索并移除不需要的博主 |
| 配置错误 | 菜单 11→4 | 统一配置管理 | 交互式检查并修复配置 |
